mermaid画时序图

发表于2021-01-13|更新于2021-01-13|mermaid
|字数总计:1.1k|阅读时长:4分钟|阅读量:

参与者（Participants）

传统时序图概念中参与者有角色和类对象之分，但这里我们不做此区分，用参与者表示一切参与交互的事物，可以是人、类对象、系统等形式。中间竖直的线段从上至下表示时间的流逝。

隐式定义

sequenceDiagram
老板->>员工:开始实行996制度
员工-->>老板: 坚决拒接

显式定义

sequenceDiagram
participant 老板
participant 员工

别名（Aliases）

sequenceDiagram
participant L as 老板
participant Y as 员工
L->>Y:开始实行996制度
Y-->>L:坚决拒绝

消息（Messages）

交互时一方对另一方的操作（比如接口调用）或传递出的信息。用单向箭头来表示——实线代表主动发出消息；虚线代表响应；末尾带「X」代表异步消息，无需等待回应。

Type Description
-> 无箭头实线
–> 无箭头虚线
->> 有箭头实线
–>> 有箭头虚线
-x 末尾带十字的实线（异步）异步消息
–x 末尾带十字的虚线（异步）异步消息 
sequenceDiagram
A->B:无箭头实线
A-->B:无箭头虚线
A->>B:有箭头实线
A-->>B:有箭头虚线
A-xB:末尾带十字的实线（异步）
A--xB:末尾带十字的虚线（异步）

激活（Activations）

从消息接收方的时间线上标记一小段时间，表示对消息进行处理的时间间隔。

使用+-来表示激活

sequenceDiagram
老板->>+员工: 公司要开始实行996制度了
员工-->>-老板: 鼓掌
老板->>+员工: 对钱没兴趣
员工-->>-老板: 鼓掌

使用activate来表示激活（typora不支持）

sequenceDiagram
A->>B: hello B
activate B
B-->>A: hello A

激活的嵌套

sequenceDiagram
A->>+B:hello B
A->>+B:hello B1
B-->>-A:hello A1
B-->>-A: hello A

注释

note left of/right of /over actor

left of

sequenceDiagram
participant 老板A
note left of 老板A: 对钱没兴趣

right of

sequenceDiagram
participant 老板B
note right of 老板B: 普通家庭

over

sequenceDiagram
participant 老板C
note over 老板C: 对996感兴趣

循环（Loops）

在时序图中使用loop来表示循环

sequenceDiagram
loop 每天上午9:00
舔狗->>女神: 早安
end

抉择（Alt）

在多个条件中作出判断，每个条件将对应不同的消息序列。相当于 if 及 else if 语句。

sequenceDiagram
alt 早上9:00
舔狗->>女神: 早安
else 晚上12:00
舔狗->>女神: 晚安
end

选项（Opt）

在某条件满足时执行消息序列，否则不执行。相当于单个分支的 if 语句。

sequenceDiagram
老板->>员工: 开始实行996
opt 用不可能
老板->>员工: 拒绝
end

并行（Opt）

将消息序列分成多个片段，这些片段并行执行。

sequenceDiagram
老板->>员工:开始实行996
par 并行
员工->>员工:刷哔哩哔哩
and
员工->>员工: 工作
and
员工->>员工: 刷朋友圈
end
员工-->>老板: 9点下班
